🎨 User Interface Layer

Description

Handles user interaction, screens, components and presentation logic.

Purpose

Provides the interface through which users interact with the system.

Folder / File Paths

  • src/components/
  • src/pages/
  • src/assets/

Responsibilities

  • Display information
  • Handle user inputs
  • Manage UI state

Relationship

Communicates with Business Logic Layer through APIs.

⚙️ Business Logic / API Layer

Description

Contains application rules, processing logic and API communication.

Purpose

Processes requests and controls application behaviour.

Folder / File Paths

  • controllers/
  • services/
  • api/

Responsibilities

  • Validate requests
  • Execute business rules
  • Generate responses

Relationship

Connects UI requests with Data Persistence Layer.
