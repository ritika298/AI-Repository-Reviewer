🖥 User Interface Layer

The UI layer handles user interaction, screen rendering, and communication with backend services.

Purpose

Provides the visual interface where users upload repositories, view analysis results, and interact with AI insights.

Folder / File Paths

  • src/components/
  • src/pages/
  • src/assets/

Responsibilities

  • User input handling
  • Displaying analysis results
  • Managing frontend components
  • Handling UI state

Layer Relationship

Communicates with the API layer to send requests and display processed repository information.

⚙ Business Logic / API Layer

The business layer contains application logic and manages communication between frontend and data services.

Purpose

Processes repository analysis requests and coordinates AI agents.

Folder / File Paths

  • controllers/
  • services/
  • api/

Responsibilities

  • Request processing
  • AI workflow execution
  • Validation logic
  • Error handling

Layer Relationship

Connects the UI layer with database and AI processing components.

🗄 Data Persistence Layer

Responsible for storing and retrieving repository information and analysis results.

Purpose

Maintains application data and repository analysis history.

Folder / File Paths

  • models/
  • repositories/
  • database/

Responsibilities

  • Database operations
  • Data management
  • Query execution

Layer Relationship

Provides stored information required by business services.

🔧 Shared Utilities & Services

Common modules used across different parts of the application.

Purpose

Improves code reuse and maintains consistency.

Folder / File Paths

  • utils/
  • helpers/
  • config/

Responsibilities

  • Common functions
  • Configuration management
  • Logging utilities

Layer Relationship

Supports UI, business logic, and data layers.
